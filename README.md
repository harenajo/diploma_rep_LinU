## Загальна інформація про проєкт

Проєкт є комп’ютерною освітньою грою у форматі візуальної новели. Основна ідея полягає в тому, щоб подати навчальний матеріал з криптографії не у вигляді окремої теорії, а через ігровий процес. Гравець поступово знайомиться з поняттями шифрування, ключа, зашифрованого повідомлення, підказки та дешифрування через сюжетні ситуації.

У центрі сюжету знаходиться студент Марк, який отримує загадкові електронні листи та стикається з невідомим застосунком AIIA. Щоб просуватися далі, гравець має читати повідомлення, аналізувати підказки, розв’язувати криптографічні задачі, відкривати заблоковані файли та приймати рішення в діалогах.

Проєкт поєднує три основні складові:

- сюжетну частину у форматі візуальної новели;
- інтерактивну частину з ноутбуком, листами, файлами та підказками;
- навчальну частину, пов’язану з базовими криптографічними поняттями.

## Сюжет гри

Головний герой Марк — студент університету. Після важкого дня він отримує загадковий файл від невідомого відправника. На перший погляд файл здається неважливим, однак після його відкриття на ноутбуці Марка з’являється невідомий застосунок AIIA.

Згодом Марк починає отримувати листи із зашифрованими повідомленнями. У цих листах містяться підказки, завдання та фрагменти старої історії, пов’язаної з університетом. Гравець поступово дізнається більше про події минулого, стосунки між персонажами та роль кожного з них у цій історії.

Проходження залежить від рішень гравця. Вибори можуть впливати на стосунки з персонажами, рівень довіри, кількість отриманої інформації та подальші сюжетні гілки.

## Структура проєкту

Основні папки Unity-проєкту:

```text
Assets/
Packages/
ProjectSettings/
```

| Папка | Призначення |
|---|---|
| `Assets/` | Основні ресурси гри: сцени, скрипти, спрайти, фони, персонажі, Ink-файли, інтерфейс та інші матеріали. |
| `Packages/` | Дані про пакети Unity, які використовуються у проєкті. |
| `ProjectSettings/` | Налаштування Unity-проєкту. |

Папки `Library`, `Temp`, `Obj`, `Logs`, `Builds` не додаються до репозиторію, оскільки вони створюються Unity автоматично або містять тимчасові файли.

## Як запустити проєкт

### 1. Встановити необхідні інструменти

Для запуску потрібні:

- Unity Hub;
- версія Unity, сумісна з проєктом;
- Git;
- Visual Studio або інший редактор C#.

### 2. Клонувати репозиторій

```bash
git clone https://github.com/harenajo/diploma_rep_LinU.git
cd diploma_rep_LinU
```

### 3. Відкрити проєкт у Unity

1. Запустити Unity Hub.
2. Натиснути `Add` або `Add project from disk`.
3. Обрати папку `diploma_rep_LinU`.
4. Дочекатися імпорту ресурсів.
5. Відкрити головну сцену гри.
6. Натиснути `Play`.

Під час першого відкриття Unity може довго імпортувати ресурси. Це нормально, оскільки папка `Library` створюється автоматично.

## Інструкція для користувача

Після запуску гри користувач проходить сюжет як візуальну новелу.

Основні дії:

1. Читати репліки персонажів.
2. Натискати кнопку переходу до наступного діалогу.
3. Обирати варіанти відповідей.
4. Взаємодіяти з кімнатою головного героя.
5. Відкривати ноутбук.
6. Читати електронні листи.
7. Відкривати файли та вводити паролі.
8. Аналізувати зашифровані повідомлення.
9. Використовувати підказки AIIA.
10. Приймати рішення, які можуть впливати на сюжет.


## Можливі проблеми та рішення

| Проблема | Рішення |
|---|---|
| Проєкт довго відкривається вперше | Дочекатися завершення імпорту ресурсів Unity. |
| Unity не бачить сцени | Перевірити, чи є потрібні сцени в папці `Assets`. |
| Не працюють діалоги | Перевірити підключення Ink JSON або Ink-файлів у компонентах сцени. |
| Не відображається персонаж | Перевірити, чи призначені спрайти персонажа в Inspector. |
| Не змінюється фон | Перевірити теги в Ink-файлі та їхню обробку у скрипті сцени. |
| Не відкривається ноутбук | Перевірити, чи підключений `LaptopController` у відповідному полі Inspector. |
| Не працює кнопка або UI-елемент | Перевірити прив’язку кнопки до потрібного методу в Inspector. |
| Після клонування немає папки `Library` | Це нормально, Unity створить її автоматично. |
